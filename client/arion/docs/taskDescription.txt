Глоссарий: 
	модель - формула, по которой производится расчет надежности элемента 

	научный режим работы программы - режим, в котором пользователю доступна для просмотра и редактирования модель

	инженерный режим работы - режим, в котором пользователю недоступна для просмотра и редактирования модель

I. Описание решаемой задачи
	1) описание задачи с точки зрения взаимодействия с сервером.
	В процессе работы программы запрос к серверу будет осуществляться в следующих ситуациях:

		а) пользователь выбрал тип элемента для добавления в модуль.
			На сервер отправляется запрос, содержащий данные о типе элемента (резистор, конденсатор, транзистор etc.). Сервер высылает перечень конечных элементов, доступных для выбора. В этот список входят реально существующие элементы и шаблон пустого элемента

			пример запроса:
			/dbPath/db?type=capacitorHF&...
			примечание: ... ~ параметры, неоходимые БД, например sortOrder, distinct etc.

			пример ответа:
			{
				"data":[
					...,
					{
						"type" : "capacitorHF",
						"id": 1234,
						"name": "к10-17а"
					},
					...
				]
			}


			б) Пользователь выбрал конечный элемент (конкретную модель (с заполненными полями характеристик) или шаблон элемента (с незаполненными полями характеристик, которые требуется заполнить вручную)).
			На сервер отпраляется запрос для получения сведений о характеристиках, присущих данному элементу и (при выборе конкретного элемента) значениях этих параметров. Также с сервера передаются сведения о том, какой тип поля ввода соответствует каждой характеристике (text, checkBox, RadioButton etc.). Для полей типа select, radioButton, checkBox etc. должен быть также указан предустановленный набор значений, доступных для выбора.
			Также, вместе с данными о характеристиках элемента с сервера приходят данные о модели для выбранного элемента.

			Пример запроса:
			/dbPath/db?type=capacitorHF&id=1234&...
			примечание: ... ~ параметры, неоходимые БД, например sortOrder, distinct etc.

			Пример ответа:
			{
				"model": "(capacity+anotherProperty)/color",
				"data":[
					...,
					{
						"type" : "capacitorHF",
						"id": 1234,
						"name": "к10-17а",
						"params":[
							{
								"paramId": "capacity",
								"paramName": "емкость",
								"paramValue": 1000,
								"editable": true,
								"description": "емкость - тут и так все понятно",
								"paramEditor": {
									"editorType": "number",
									"editorPlaceHolder": "пФ",
									"editorMinValue": 0,
									"editorMaxValue": 1000000
								}
							},
							{
								"paramId": "color",
								"paramName": "цвет",
								"paramValue": "red",
								"editable": true,
								"description": "этот параметр является самым важным",
								"paramEditor": {
									"editorType": "select",
									"editorOptions":[
										"red",
										"green",
										"blue"
									]
								}
							}
						]
					},
					...
				]
			}

			Примечание: значение поля paramValue может принимать значение NULL.

	2) описание с точки зрения GUI
		видимая область разделена на три функциональных зоны
		1. область выбора
		2. область редактирования и расчетов
		3. область построения графиков и диаграмм

		1. область выбора
			Изначально содержит область, в которой будет строиться дерево модулей и элементов, и кнопки "Добавить модуль", "Добавить элемент" и "Удалить узел", расположенные над деревом элементов. Уровни вложенности дерева определяются количеством отступов от левого края. Соответственно, чем большая вложенность узла, тем большее количество отступов. Все узлы дерева кликабельны. При выборе узла-элемента в области редактирования отображаются характеристики элемента. Для модулей характеристики не отображаются. 

			Поведение кнопок:
			а) фокус на элементе: 
				"Добавить модуль" - на одном уровне вложенности с выбранным элементом добавляется новый модуль
				"Добавить элемент" - на одном уровне вложенности с выбранным элементом добавляется новый элемент
				"Удалить узел" - удаление выбранного элемента
			б) фокус на модуле:
				"Добавить модуль" - новый модуль добавляется в качестве вложенного в данный модуль
				(опционально: отображать диалоговое окно, предоставляющее выбор: добавлять в качестве вложенного или на одном уровне с выбранным модулем)
				"Добавить элемент" - новый элемент добавляется в качестве вложенного в данный модуль
				(опционально: отображать диалоговое окно, предоставляющее выбор: добавлять в качестве вложенного или на одном уровне с выбранным модулем)
				"Удалить узел" - удаление выбранного элемента
				(опционально: отображать диалоговое окно, требующее подтверждение удаления выбранного модуля со всеми его потомками)

			Дополнительный функционал (точки расширения. не являются частью базовой версии): 
			а) сортировка элементов и модулей по алфавиту/другому критерию
			б) сворачивание дерева по принципу accordion
			в) поиск элемента по названию
			г) drag-n-drop элементов и модулей с переформированием структуры дерева
			д) возможность copy-paste для элементов/модулей/групп элементов


		2. Область редактирования и расчетов
		Содержит три вкладки:
			1. Вкладка редактирования характеристик
			2. Вкладка редактирования модели и расчетов
			3. Вкладка статистических расчетов 

			1. Вкладка редактирования характеристик
			Содержит таблицу размером 2хN, где N - число доступных для выбранного элемента характеристик
			В левой колонке отображается название характеристики (+единица измерения?). Каждая ячейка правой колонки представляет собой поле ввода. Может быть представлена различными типами полей ввода: number, text, select, checkBox, range etc. Значения этих полей либо настраиваются пользователем, либо устанавливаются на основе данных, пришедших с сервера (в случае выбора реально существующего элемента)

			Также вкладка содержит кнопку "Рассчитать", по нажатии на которую происходит расчет параметров компонента по модели и построение графика в области построения графиков и диаграмм (вывод результата в текстово-числовой интерпретации?)

			2. Вкладка редактирования модели и расчетов
			Горизонтально делится на три части. 
			верхняя часть содержит кнопки "Изменить модель", "Сбросить модель" и "Рассчитать", а также область отображения характеристик элемента, участвующих в формировании модели. При нажатии на кнопку "Изменить модель" появляется диалоговое окно с полем типа text, в которое вписана модель для расчета надежности. Пользователь может редактировать модель и сохранить её.  При нажатии на кнопку "Сбросить модель" модель, измененная пользователем будет возвращена к первоначальному состоянию. по нажатии на кнопку "рассчитать" будет произведен расчет по модели. Результаты расчета и промежуточные вычисления будут отображены в логе

			Центральная часть представляет собой текстовое поле, недоступное для редактирования. Содержит в себе значения всех характеристик элемента с описаниями. Также содержит в себе используемую в данный момент времени модель.

			Нижняя часть предназначена для отображения лога вычислений.
			Содержит сведения о входных параметрах, промежуточные вычисления (?) и результат вычилсений.
